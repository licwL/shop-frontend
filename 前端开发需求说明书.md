# 前端开发需求说明书

## 基础信息

- **后端**：`http://localhost:8080`，Vite proxy 转发，前端发相对路径
- **鉴权**：登录返回 token → 存 localStorage → 请求头 `token` 字段自动携带 → 401 跳登录
- **响应格式**：`{ code: 1, data: {...} }` 成功；`{ code: 0, msg: "..." }` 失败
- **分页格式**：参数 `page`+`pageSize`，返回 `{ total: 100, records: [...] }`
- **主题**：`shared/styles/variables.scss`，现代简约品质感
- **公共**：`shared/request.js`、`shared/utils.js`、`shared/components/Pagination.vue`

## 管理端（8 个页面）

### 文件上传

所有需要上传图片的地方，调用 `POST /admin/common/upload`（multipart/form-data，字段名 `file`），返回的 `data` 就是图片 URL。

---

### 1. 登录页 `/login`

- 用户名密码登录 → `POST /admin/employee/login`
- 入参 `{ username, password }`，返回 `{ id, userName, name, token }`
- 存 token 到 localStorage，跳转首页

### 2. 仪表盘 `/dashboard`

- 用 ECharts
- **4 个统计卡片**：今日营业额、有效订单数、订单完成率、平均客单价、新增用户数
- **营业额趋势折线图**：`GET /admin/report/turnover?begin=&end=`
  - dateList 是 X 轴（逗号分隔的日期），turnoverList 是 Y 轴（逗号分隔的金额）
- **订单统计折线图**：`GET /admin/report/order?begin=&end=`
  - dateList → X 轴，orderCountList + validOrderCountList → 两条线
- **销量 Top10 柱状图**：`GET /admin/report/top10?begin=&end=`
  - nameList → X 轴，numberList → Y 轴
- 底部显示汇总：totalOrderCount、validOrderCount、orderCompletionRate
- 概览数据用 `GET /admin/report/business?begin=&end=`

### 3. 商品管理 `/product`

- **表格列**：ID、商品名、分类（categoryName）、价格、库存、销量、状态（上架/下架标签）、创建时间
- **搜索栏**：商品名输入框 + 分类下拉框（`GET /admin/category/list` 获取选项）+ 状态下拉
- **新增/编辑弹窗**：商品名、分类下拉、价格、库存、图片上传、描述 textarea、状态开关
- **操作**：每行编辑/删除；顶部批量删除按钮；顶部新增按钮
- **API**：
  - 分页列表 `GET /admin/product/page?page=&pageSize=&name=&categoryId=&status=`
  - 新增 `POST /admin/product`，请求体 `{ name, categoryId, price, image, description, stock, status }`
  - 修改 `PUT /admin/product`
  - 批量删除 `DELETE /admin/product?ids=1&ids=2`
  - 查单个 `GET /admin/product/{id}`

### 4. 订单管理 `/order`

- **表格列**：订单号、用户名、金额、状态标签（用颜色区分）、收货人、下单时间
- **搜索栏**：订单号输入 + 状态下拉 + 时间范围选择器
- **详情**（抽屉或弹窗）：订单信息 + 商品明细列表（从 `orderDetailList` 渲染）
- **操作**：状态为"已付款"→ 显示"发货"按钮；状态为"已发货"→ 显示"完成"按钮
- **状态说明**：1=待付款（灰色）、2=已付款（蓝色）、3=已发货（橙色）、4=已完成（绿色）、5=已取消（红色）
- **API**：
  - 分页 `GET /admin/order/page?page=&pageSize=&number=&status=&beginTime=&endTime=`
  - 状态变更 `PUT /admin/order/status/{status}?id=`，status: 3=发货, 4=完成
  - 详情 `GET /admin/order/details/{id}`

### 5. 用户管理 `/user`

- **表格列**：ID、姓名、手机号、性别、状态（启用/禁用标签）、注册时间
- **搜索栏**：姓名输入 + 手机号输入
- **操作**：每行"启用/禁用"按钮
- **API**：
  - 分页 `GET /admin/user/page?page=&pageSize=&name=&phone=`
  - 启用禁用 `POST /admin/user/status/{status}?id=`，status: 1=启用, 0=禁用

### 6. 通知管理 `/notice`

- **表格列**：标题、类型名称（typeName）、状态（草稿/已发布标签）、创建时间
- **搜索栏**：标题输入 + 类型下拉（`GET /admin/notetype/list` 获取选项）
- **新增/编辑弹窗**：标题、内容 textarea、类型下拉、状态开关（草稿/发布）
- **操作**：批量删除、每行编辑/发布/撤回
- **API**：
  - 分页 `GET /admin/notice/page?page=&pageSize=&title=&typeId=&status=`
  - 新增 `POST /admin/notice`，请求体 `{ title, content, typeId, status }`
  - 修改 `PUT /admin/notice`
  - 批量删除 `DELETE /admin/notice?ids=1&ids=2`
  - 发布撤回 `PUT /admin/notice/status/{status}?id=`，status: 1=发布, 0=撤回

### 7. 分类管理 `/category`

- **表格列**：ID、名称、排序、状态标签
- **操作**：新增/编辑/删除按钮；启用/禁用按钮
- **API**：
  - 分页 `GET /admin/category/page?page=&pageSize=&name=`
  - 新增 `POST /admin/category`，请求体 `{ name, sort }`
  - 修改 `PUT /admin/category`
  - 删除 `DELETE /admin/category?id=`
  - 启用禁用 `POST /admin/category/status/{status}?id=`
  - 全部列表 `GET /admin/category/list`

### 8. 通知类型 `/notetype`

- 入口可放在通知管理页内或独立导航
- **表格列**：ID、名称
- **操作**：新增弹窗（仅名称）、每行编辑/删除
- **API**：POST/PUT/DELETE /admin/notetype，GET /admin/notetype/list

### Excel 导出

- 订单导出：`GET /admin/export/orders`（直接下载 .xlsx 文件流）
- 商品导出：`GET /admin/export/products`

## 管理端布局

- **侧边栏**：深色背景，菜单 8 项（含仪表盘、商品、订单、用户、通知、分类、通知类型），底部退出
- **顶栏**：平台名称 + 当前用户名 + 退出按钮
- **内容区**：面包屑 + 页面主体
