# 青蔬市集 — 前端开发需求说明书

## 基础信息

- **后端**：`http://localhost:8080`（通过 Vite proxy 转发，前端发相对路径即可）
- **鉴权**：登录返回 token，请求头 `token` 字段携带；401 时自动跳登录页
- **统一响应**：`{ code: 1, data: {...} }` 成功；`{ code: 0, msg: "..." }` 失败
- **分页**：参数 `page` + `pageSize`，返回 `{ total: 100, records: [...] }`
- **主题**：绿色系精品生鲜电商，`shared/styles/variables.scss`，氛围清新有质感
- **公共代码**：`shared/request.js`（axios）、`shared/utils.js`（工具函数）、`shared/components/Pagination.vue`（分页）

## 一、管理端 — 页面清单（共 8 个页面）

### 1. 登录页 `/login`

- **功能**：用户名 + 密码登录
- **API**：`POST /admin/employee/login`，入参 `{ username, password }`，返回 `{ id, userName, name, token }`
- **交互**：登录成功存 token 到 localStorage，跳转首页；失败弹错误信息

### 2. 仪表盘 `/dashboard`

- **功能**：数据概览，今日关键指标
- **内容**：
  - 4 个统计卡片：今日营业额、有效订单数、订单完成率、平均客单价、新增用户数
  - 营业额趋势折线图（日期 → 金额）
  - 订单统计折线图（日期 → 订单数/有效订单数）
  - 销量 Top10 柱状图（商品名 → 销量）
- **API**：
  - `GET /admin/report/business?begin=2026-07-04&end=2026-07-04` → 概览数据
  - `GET /admin/report/turnover?begin=2026-06-27&end=2026-07-04` → 营业额趋势
  - `GET /admin/report/order?begin=2026-06-27&end=2026-07-04` → 订单趋势
  - `GET /admin/report/top10?begin=2026-06-27&end=2026-07-04` → 销量排行
- **图表**：使用 ECharts 或 Element Plus 兼容的图表库

### 3. 商品管理 `/product`

- **功能**：商品的增删改查
- **列表**：表格展示（ID、名称、分类、价格、库存、销量、状态、创建时间）
- **搜索**：顶部搜索栏 — 商品名输入框 + 分类下拉框 + 状态下拉框 + 搜索按钮
- **操作**：新增按钮 → 弹窗表单；每行编辑/删除按钮；批量删除
- **新增/编辑弹窗**：商品名、分类（下拉）、价格、库存、图片 URL、描述、状态（上架/下架）
- **API**：见 `docs/API接口文档-管理端.md` 第 3 节商品管理（5 个接口）

### 4. 订单管理 `/order`

- **功能**：查看和管理所有订单
- **列表**：表格展示（订单号、用户名、金额、状态标签、收货人、下单时间）
- **搜索**：订单号输入 + 状态下拉（待付款/已付款/已发货/已完成/已取消）+ 时间范围
- **操作**：查看详情；待发货订单 → 发货按钮；已付款订单 → 完成按钮
- **详情**：弹窗或抽屉展示订单基本信息 + 商品明细列表
- **API**：见文档第 4 节订单管理（3 个接口）

### 5. 用户管理 `/user`

- **功能**：查看注册用户，启用/禁用
- **列表**：表格展示（ID、姓名、手机号、性别、状态、注册时间）
- **搜索**：姓名输入 + 手机号输入
- **操作**：每行启用/禁用按钮
- **API**：见文档第 5 节用户管理（2 个接口）

### 6. 通知管理 `/notice`

- **功能**：通知的增删改查 + 发布/撤回
- **列表**：表格展示（标题、类型、状态(草稿/已发布)、创建时间）
- **搜索**：标题输入 + 类型下拉
- **新增/编辑弹窗**：标题、内容（富文本或 textarea）、类型下拉、状态（草稿/发布）
- **操作**：批量删除；每行发布/撤回按钮
- **API**：见文档第 6-7 节通知管理（10 个接口，含通知类型管理）

### 7. 分类管理 `/category`

- **功能**：商品分类的增删改查
- **列表**：表格展示（ID、名称、排序、状态）
- **操作**：新增/编辑/删除；启用/禁用
- **API**：见文档第 2 节商品分类管理（6 个接口）

### 8. 通知类型 `/notetype`

- **功能**：通知类型的增删改查（入口可放在通知管理页或导航栏）
- **列表**：表格展示（ID、名称）
- **操作**：新增按钮 → 弹窗输入名称；每行编辑/删除

## 二、管理端 — 布局要求

- **侧边栏导航**：左侧固定，深色背景，菜单项 8 个（上述页面 + 退出）
- **顶栏**：显示平台名称「青蔬市集 · 管理后台」+ 当前用户名 + 退出按钮
- **内容区**：右侧自适应，面包屑导航 + 页面内容

## 四、开发顺序

1. 先做管理端（admin/），8 个页面从上到下逐个完成
2. 公共代码放 shared/，第二个页面发现重复就立刻抽
3. 管理端全部完成后，再开发用户端（user/）
